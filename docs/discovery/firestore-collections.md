## 目的

旧 Firestore のコレクション構造を v2 の PostgreSQL スキーマ設計（#15）に
1:1 対応できる粒度で文書化する。**機微情報（実ユーザー名・実校データ）は書かない**。
構造のみ。

## 必要な記載項目（テンプレート）

各コレクションごとに以下を埋める:

### `<collection_name>`

- **目的**: 何を保存しているか（1〜2行）
- **ドキュメント ID 規則**: 自動生成 / 業務キー / 複合キーなど
- **親コレクション**: トップレベル / `<parent>/{id}/<sub>` の形
- **想定ドキュメント数（オーダー）**: 100 / 10k / 1M など
- **フィールド一覧**:

  | フィールド | 型 | 必須 | 説明 | 備考 |
  |---|---|---|---|---|
  | `school_id` | string | ✅ | テナント分離キー | v2 では RLS の `app.current_school_id` に対応 |
  | ... |
- **インデックス**: `firestore.indexes.json` から該当複合インデックスを抜粋
- **`firestore.rules` の該当ブロック**: read/write の許可条件を要約
- **書き込み元**: クライアント / Cloud Functions / 移行スクリプト
- **PII 含有**: あり/なし（あれば項目名を列挙）
- **v2 移行先テーブル案**: `packages/db/schema/*.ts` の予定テーブル名

## 移行・廃止判断

各コレクションに対して:

- `migrate`: v2 で同等機能あり、データ移行する
- `transform`: スキーマを変えて移行する
- `drop`: v2 では廃止
- `archive`: 読み取り専用で保管のみ

の判断を `scripts/migration/` の設計（後続 issue）に渡せる粒度で記録する。
