## 1. 目的とスコープ（in / out）

統合された staging に対し、**3 つの利用文脈（サイネージ TV / 教員管理画面 / 生徒スマホ）でプロダクトが
「実際に使えること」**を、要件トレーサブルに確認する受入ゲート。親戦略トラック②の詳細化であり、
PR 単位の component テストの二重化ではなく、**統合画面に対する横断的・実ブラウザ検証**を担う。

### in（このトラックで見る）
- アクセシビリティ（WCAG 2.2 AA、トレース元 [NFR05](../../requirements/non-functional/NFR05-accessibility.md)）。
- ユーザビリティ・操作性（タスク完遂、キーボード単独操作、エラー回復、即公開判断のしやすさ）。
- GUI 視覚回帰・レイアウト崩れ（フォント/コントラスト/オーバーフロー）。
- レスポンシブ・マルチデバイス（サイネージ大画面、PC/タブレット、スマホ縦横）。

### out（このトラックで見ない）
- 機能の業務的正しさ（時間割マージ結果の妥当性など）→ トラック①（機能受入）。
- 認可・テナント越境・PII 漏洩・プロンプトインジェクション → トラック③（セキュリティ）。
  - ただし UI 上で **色のみに依存した秘匿/可視判定** など a11y 起因の情報伝達欠陥は本トラックで拾う。
- 応答速度の数値 SLA（初期ロード < 1.5s 等の計測）→ トラック④（非機能）。本トラックは体感上の破綻のみ。
- **実機サイネージ・実生徒端末・本番環境での確認** → 人間 / 導入フェーズ（第 9 節）。

## 2. 前提・環境・3つの利用文脈

- 環境は **staging 限定・合成データのみ**。実生徒 PII・本番投入は禁止（CLAUDE.md ルール4 / セキュリティ最優先）。
- 認証経路: 教員/管理者は Identity Platform ログイン（`/login`）→ `/admin/*`。生徒は magic-link（`/s/{token}` → `/student`）。
  サイネージは **匿名公開**（`/signage/{classToken}`、middleware 除外・token+RLS でスコープ）。
- 本プロジェクトは **Claude in Chrome / Preview MCP の利用が許可済み**。これらを GUI ウォークスルーの
  主力手段として組み込む（第 6 節）。

| # | 文脈 | 代表ルート | デバイス前提 | 操作様式 | 主な a11y 圧力 |
|---|---|---|---|---|---|
| 1 | **サイネージ TV** | `/signage/{classToken}` | 大画面・遠距離（教室後方）・無人・常時表示 | 受動表示（入力なし） | 最小フォント 24px・遠距離コントラスト・`prefers-reduced-motion`・last-good 維持の視認 |
| 2 | **教員管理画面** | `/login`, `/admin/editor/{classId}`, `/admin/school`, `/admin/system/*` | PC / タブレット | 編集・即公開判断（キーボード多用） | キーボード単独操作・フォーカス可視・エラー伝達が色依存でない・即公開の可逆性が分かる |
| 3 | **生徒スマホ** | `/s/{token}`, `/student`, `/guide` | スマホ（縦持ち中心） | 片手タップ・音声入力 | タップ領域 44pt・縦横両対応・音声入力配慮・読み上げ |

## 3. テスト観点の分類

| 観点 | 名称 | 内容 | 主トレース |
|---|---|---|---|
| **A** | アクセシビリティ（WCAG 2.2 AA） | コントラスト、キーボード、フォーカス可視、セマンティック HTML / aria、色非依存、reduced-motion | NFR05 |
| **B** | ユーザビリティ・操作性 | タスク完遂率、導線の分かりやすさ、エラー文言・回復、即公開の安心感、空状態/失効状態の伝達 | NFR05（生徒 UI）+ 各 F 要件 |
| **C** | GUI 視覚回帰・レイアウト | スクショ基準との差分、フォント崩れ、オーバーフロー/折り返し、バッジ・ドット等の描画 | NFR05（サイネージ表示） |
| **D** | レスポンシブ・マルチデバイス | ブレークポイント、縦横回転、大画面/小画面でのグリッド再配置、タッチターゲット | NFR05（生徒 UI / サイネージ） |

> A は自動（axe-core / Lighthouse）+ 手動（スクリーンリーダー）の併用が必須。B/D は Claude 実ブラウザ
> ウォークスルーが主力。C は Playwright スクショ基準が主力。役割分担は第 6 節。

### 拡張方針（網羅でなく増殖規則）
- A: axe-core ルールセット（wcag22aa タグ）を全公開ルートに自動適用し、上表は手動 SR / 目視で深掘りする代表点のみ列挙。
- B: 文脈ごとに「主要 1 タスクの完遂シナリオ」を 1 本以上常設し、機能追加時に追補。
- C: 各文脈の代表ビューにスクショ基準を持ち、新ページ追加時に基準を 1 枚追加。
- D: ブレークポイント（360 / 768 / 1024 / 1920 / 縦置きサイネージ）を固定マトリクスとし、新ビューを横断適用。

## 5. 合否基準

### WCAG 違反の重大度別扱い（A 観点）
| 重大度（axe impact / WCAG レベル） | 扱い |
|---|---|
| **critical / serious かつ AA 必須**（コントラスト、キーボードトラップ、フォーカス不可視、名前無し操作子） | **ブロッカー**。Exit 不可。修正 PR → Reviewer → 再検証で閉じる |
| **moderate**（見出し階層の乱れ、冗長な aria 等） | 原則修正。導入前に直せない正当理由があれば defect-log に記録 + 暫定回避 + フォロー issue 必須 |
| **minor / best-practice**（AAA 相当、推奨レベル） | 記録のみ。Exit は阻害しない |

### 視覚回帰の許容差（C 観点）
- ピクセル差分しきい値: **0.1%（1000px あたり 1px 相当）以内**を pass。
- アンチエイリアス/フォントレンダリング由来の微差は許容、**レイアウトシフト・文字欠け・はみ出し・色崩れは不許容**（しきい値内でも fail 判定）。
- 基準スクショの更新は意図的変更時のみ、PR で差分を Reviewer に提示して承認。

### 必須クリア項目（go の前提、すべて満たすこと）
1. **3 文脈 × axe-core wcag22aa が critical / serious ゼロ**（公開全ルート）。
2. **キーボードのみで各文脈の主要タスクを完遂可能**（教員: 編集→公開、生徒: 着地→主要操作、サイネージは入力なしのため対象外）。
3. **サイネージ本文フォント 24px 以上・テキストコントラスト 4.5:1 以上**を実測で確認。
4. **生徒スマホのタップ領域 44pt 以上・縦横両対応**を確認。
5. **`prefers-reduced-motion` 尊重**（広告自動再生/動きの抑制）。
6. **色のみに依存した情報伝達がない**。
7. 手動スクリーンリーダー確認で **主要 3 画面（サイネージ盤面 / 教員エディタ / 生徒着地）が破綻なく読み上げ**られる。

> **設計時点の既知ギャップ（UX-003 reduced-motion 未尊重 / UX-008 ログインエラー色依存）の扱い**:
> 両者は [defect-log.md](../defect-log.md) P-1 / P-2 に先取り記録済み。実行時に axe-core 重大度を確定し、
> 次の分岐で go 経路を**循環させない**:
> - **項目 5/6 に直結する critical / serious 違反**として確定 → **ブロッカー**。in-track 修正 PR → Reviewer → 再検証で閉じてから go（必須）。
> - **moderate 以下**と確定、かつ修正が本トラック外（別 F / issue）に切り出される → **go-with-conditions**（defect-log + 暫定回避 + フォロー issue を go/no-go レポートに明記）。
> これにより「既知ギャップ＝無条件 no-go」という構造的デッドロックを避け、重大度に応じた判定とする。

> go/no-go: 上記 7 項目すべて満たし（既知ギャップは上記分岐で解決済）、moderate 残件が defect-log で追跡されていれば **go**。
> critical / serious が 1 件でも未修正なら **no-go**。

## 6. 手法・ツール（役割分担）

| レイヤ | ツール | 役割 | 担い手 |
|---|---|---|---|
| **自動 a11y（広く浅く）** | **axe-core** | 全公開ルートで wcag22aa ルール機械チェック。Playwright E2E（`@axe-core/playwright`）に組込み CI 常設（NFR05 受入条件）。`packages` 既存スイートと別に検証用シナリオへ拡張 | Claude（CI 組込 + 検証時実行） |
| **自動スコア** | **Lighthouse** | Accessibility / Best-Practices スコア、コントラスト・タップターゲットの一次検出。文脈別ビューポート（モバイル/デスクトップ）で実行 | Claude |
| **実ブラウザ ウォークスルー（深く操作）** | **Claude in Chrome / Preview MCP** | 主力手段。実際に staging を開き、キーボード単独操作・タスク完遂（UX-009 等）・縦横回転・失効画面（UX-005/017）・薄字目視を**人間の代わりに操作して観察**。スクショ取得とコンソール/ネットワーク確認も担う | Claude |
| **視覚回帰** | **Playwright スクリーンショット**（`toHaveScreenshot`） | 文脈別代表ビューの基準スクショ差分。第 5 節の許容差で判定 | Claude |
| **手動スクリーンリーダー（最終確認）** | **NVDA（Windows）/ VoiceOver（mac/iOS）** | 自動ツールで捕まらない読み上げ順・文脈・ライブリージョンの妥当性。**Phase 後半の最終確認**として人手で実施 | 人間（Claude は手順書とチェックリストを用意） |

役割分担の原則:
- **axe-core / Lighthouse** = 機械判定できる違反を網羅的に拾う一次フィルタ（広く浅く、CI 常設）。
- **Claude 実ブラウザ操作** = 自動ツールが判定できない「操作の通しやすさ・見え方」を実体験で検証（深く、Phase 検証時）。
- **手動 SR** = 最終的な質的確認。自動 + Claude を通過した主要画面に限定して人手コストを最小化。
- 検出はすべて [defect-log.md](../defect-log.md) に集約 → 修正 PR → Reviewer（別 spawn）→ 再検証でクローズ（busy CEO 自律 merge 範囲）。

## 8. 成果物・記録

- **axe レポート**: 文脈 × ルート別の違反一覧（impact / WCAG criterion / 対象セレクタ）。CI artifact として保存。
- **Lighthouse スコア**: モバイル/デスクトップの Accessibility・Best-Practices スコアと指摘内訳。
- **スクリーンショット**: 各文脈の代表ビュー基準 + 差分検出時の比較画像（合成データのみ。PII を含めない）。
- **ウォークスルー記録**: Claude in Chrome / Preview MCP のタスク完遂ログ（手順・観察・到達可否）。
- **手動 SR チェックリスト**: NVDA / VoiceOver 用の読み上げ確認項目と結果。
- **defect-log への登録**: 検出欠陥 → 重大度 → 修正 PR / フォロー issue 追跡（[defect-log.md](../defect-log.md)）。
- 上記を集約し go/no-go 判断データとして [go-no-go-report.md](../go-no-go-report.md) へ要約を提供。

## 9. Claude / 人間の境界

| 区分 | 内容 |
|---|---|
| **Claude 主導** | axe-core / Lighthouse / Playwright 視覚回帰の自動化・CI 組込、Claude in Chrome / Preview MCP による実ブラウザ ウォークスルー、検出 → 修正 PR → Reviewer → 再検証ループ、手動 SR の手順書/チェックリスト整備。すべて **staging・合成データ**上。 |
| **人間 / 導入フェーズに残す** | ① **実機サイネージでの遠距離視認確認**（実 TV・教室後方からの実視認、輝度/映り込み）、② **実生徒端末での片手操作・音声入力の実機確認**（多様な端末/OS/支援技術）、③ **手動 NVDA / VoiceOver の最終レビュー**（質的判断）、④ 本番環境・実 PII に対する一切の UI 検証。 |

> Claude の実ブラウザ操作はエミュレート（ビューポート/デバイスエミュレーション）であり、**実機の物理的視認性
> （遠距離・輝度・タッチの当たり）は代替しない**。ここが Claude/人間の本質的な境界。
