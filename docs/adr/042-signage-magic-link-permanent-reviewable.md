# ADR-042: サイネージ/生徒アクセス magic-link を無期限・再表示可に統一する（kind 分離なし）

- 状態: Accepted（2026-06-18、ユーザー判断・実装前）
- 日付: 2026-06-18
- 関連: [ADR-019 (RLS 二層)](019-rls-two-layer-tenant-isolation.md), [ADR-041 (system_admin 特定校スコープ操作)](041-system-admin-school-scoped-operations.md), [CLAUDE.md ルール1/2/4/5](../../CLAUDE.md), F05（生徒アクセスリンク）, F15（TV デバイス）, #863（旧: リンク期限を1年に是正）, [[reference_kimiteras_tv_connect_and_deploy]]

## 文脈

`magic_links` の 1 トークンが **2 用途を兼ねている**:

- `https://app.school-signage.net/signage/<token>` — **サイネージ盤面**。`tv_devices.signage_url` に埋め込み、教室モニタが WebView で常時表示する。
- `https://app.school-signage.net/s/<token>` — **生徒の匿名 read**。同じクラスの盤面（予定/連絡/生徒呼び出し/来校者等）を生徒が見る。

現状の設計（秘匿前提）:

1. `magic_links` は **`token_hash` のみ保存**（平文は発行レスポンスのみ・再表示不可。ルール5系「平文トークンは発行時のみ」）。
2. **有効期限あり**（schema 既定 90 日 / API は #863 で **1 年**に設定）。
3. **失効**は `revoked_at` で可能。
4. 解決は `resolve_magic_link()`（SECURITY DEFINER）が期限・失効を確認して `{schoolId, classId}` を返す。

しかし**サイネージは「公開・常時・広告モデル」**であり、この秘匿前提と運用が噛み合わない:

- 常時稼働モニタは**期限切れで盤面が消える**（1 年ごとに全モニタ再発行が必要）。
- 発行 URL が**その場限り**のため、後からモニタの URL を確認/再設定できず、運用が重い。
- モニタ新規追加は **学校作成→クラス作成→magic-link 発行（その場限り）→URL コピー→tv-device 作成で URL 貼付→device_id コピー→端末プロビジョン** とページ往復＋不透明トークンの手コピペで**非常に面倒**（ユーザーが UI で実体験）。

ユーザー（運営）は、サイネージ運用に合わせてこの前提を緩める判断をした。論点ごとの判断を本 ADR に記録する。

## 決定

### D1. 有効期限を「無期限」許容にする

`magic_links.expires_at` を **NULL 許容**にし、**NULL = 無期限**として `resolve_magic_link()` が通す（`expires_at IS NULL OR now() < expires_at`）。発行 API は既定で無期限（NULL）で発行する。

- **根拠**: 盤面は公開前提・広告モデル。漏洩しても実害が無く、むしろ閲覧者増＝impression 増で広告価値が上がる、というユーザー判断。常時モニタが期限切れで黒画面化する運用負債を断つ。

### D2. 発行 URL を「再表示可」にする（平文トークン保存）

発行済みトークンの**完全な URL を後から再表示/コピーできる**ようにする。これには平文トークンの保存が必要なので、`magic_links` に **平文トークン列**（例 `token`）を追加して保存する。

- **再表示できる人**: **system_admin + 学校管理者（school_admin・自校分、RLS スコープ）**。
- **根拠（秘匿緩和の正当化）**: 再表示は運用者本人（運営・自校管理者）に限られ、配布相手に二度と見せない規律とは別物。なお `tv_devices.signage_url` には**既に平文トークンが格納**されており、平文が DB に載ること自体は現状と同じ（新規リスクではない）。

### D3. 秘匿・PII 抑制をしない（意図的な方針緩和・要記録）

生徒配布リンク（`/s/`）も**無期限・秘匿不要**でよい（ユーザー明言）。**生徒呼び出し/来校者の氏名が出る盤面も、公開リンク経由で第三者が見られる状態を許容**する（PII ブロックの抑制・学校別 ON/OFF は**実装しない**）。

- **⚠ これは従来の秘匿/PII 姿勢（ルール4・「漏洩したらサービス終了」）を意図的に緩める決定である**。盤面は物理的に公開済みという前提、かつ token は 128bit 級の推測不能値である一方、**無期限・公開リンクの漏洩時は生徒氏名が無制限に外部露出しうる**。ユーザーは情報を理解の上で許容した。
- **要対応（ブロッカーではない）**: 一般提供（GA）前に、**個人情報保護法/文科省 GL との整合を所管で一度確認**することを推奨する。本 ADR はこの緩和が「見落とし」でなく**意図的決定**であることを記録する。

### D4. 手動の失効・再発行は維持する

無期限化しても、運営/自校管理者が**手動で失効（`revoked_at`）→ 再発行**できる導線は残す（漏洩時/差し替え時の安全弁）。再発行時は、そのトークンを使う `tv_devices.signage_url` も新トークンへ更新する（孤児化防止）。

### D5. トークンの kind 分離はしない

当初検討した「サイネージ用 / 生徒配布用でトークンを `kind` 分離し別ポリシー（公開↔秘匿）にする」案は**採用しない**。D3 で生徒配布も秘匿不要と決まったため分離の必要が消えた。**1 クラス = 1 トークン**を無期限・再表示可・失効可で両用途に共用する（最小実装）。

### D6. モニタ追加フローを「クラス選択」化する

`/ops/tv-devices/new`（および編集）で、サイネージ URL を手貼りする代わりに **学校→クラスを選ぶ**だけにする。サーバが当該クラスの magic-link トークンを**自動発行（無ければ）/ 再利用（あれば）**して `signage_url`（`?design=patternN` 付き）を組み立て、ラベルもクラス名から既定補完する。D1/D2 により既存トークンを引けるので、magic-link ページへの寄り道と手コピペが消える。

- 手貼り（任意 URL 直接入力）も後方互換で残す（既存運用・特殊ケース）。

## 影響

- **DB（migration）**: `magic_links` に `expires_at` NULL 許容化 + 平文トークン列追加。`resolve_magic_link()`（SECURITY DEFINER）を NULL 期限許容に改修。
  - ⚠ **本番適用は人間専任**（migrator ロール）。`resolve_magic_link` を誤ると**全サイネージ/生徒リンクが静かに全断**するため、staging 検証後に人手で適用する（[[apply-migration]] / db-migrations runbook）。
- **API**: 発行（無期限既定・平文保存）/ 再表示（system_admin + 自校 school_admin）/ 失効・再発行（signage_url 更新）。
- **UI**: `MagicLinkManager`（再表示・「その場限り」文言撤去・無期限表示・失効維持・school_admin 自校可視）/ tv-device 作成・編集フォーム（クラス選択）。
- **後方互換**: 既存の有効期限つきトークンはそのまま有効（NULL でない expires_at は従来どおり期限判定）。手貼り URL も残す。
- **PR 分割（ルール6: 1PR≤500行）**: ①DB+resolve ②発行/再表示/失効 API + MagicLinkManager ③tv-device フォームのクラス選択化。

## 代替案

- **kind 分離（signage / student で別ポリシー）**: D3 で student も秘匿不要となり不要（不採用）。将来 student の秘匿要件が復活したら本 ADR を supersede して分離を再検討。
- **平文を暗号化保存（Secret Manager / app 層暗号）**: 秘匿不要の判断下では過剰。平文列で十分・運用簡素（不採用）。
- **無期限でなく超長期有限（例 10 年）**: 結局期限切れ運用が残る。無期限（NULL）が運用に素直（不採用）。
- **PII ブロックを公開トークンで抑制 / 学校別 ON-OFF**: ユーザーが「そのまま許容」を選択したため今回は実装しない（将来 D3 の方針が変われば再訪）。
