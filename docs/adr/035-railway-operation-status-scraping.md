# ADR-035: 鉄道運行情報（笠松駅・名鉄）の外部取得 — 公式ページのスクレイピングと端末閉域の維持

- 状態: Accepted（2026-06-10、ユーザー判断）
- 日付: 2026-06-10（Proposed / Accepted 同日。パターン2 盤面の「鉄道」ウィジェット実装と同時）
- 関連: [ADR-021 サイネージ天気（閉域パターンの先例）](021-weather-data-source-jma.md), [ADR-009 Terraform / 単一 egress](009-terraform.md), [ADR-019 RLS 二層 + 公開参照マスタ特例], [CLAUDE.md ルール5（secret/log）], [docs/STATUS.md「外部システム自動取込みは現フェーズで実装しない（2026-05-28 ユーザー判断）」], パターン2 鉄道（railway_status・本スライス + 取得 Job 別 PR）

## 文脈

パターン2 のサイネージ盤面に、最寄りの **笠松駅（名鉄 名古屋本線・竹鼻線。岐南工業の最寄）** の運行情報を出したい（ユーザー要望）。

- **名鉄には「無料で使える公式リアルタイム運行情報 API」が実質存在しない**。ODPT（公共交通オープンデータ）は首都圏事業者が中心で、名鉄の運行情報は対象外。
- STATUS.md に「外部システム自動取込みは現フェーズで実装しない（閉域維持・セキュリティ優先、2026-05-28 ユーザー判断）」と明記済み。本機能はこれを**運行情報の 1 点に限って反転**する（ユーザー確定 2026-06-10）。

## 候補

| 候補 | 概要 | 評価 |
|---|---|---|
| A. 商用運行情報 API（駅すぱあと / NAVITIME 等） | 名鉄カバー・確実・規約クリア | **有料**（契約 + 月額 + API キー）。MVP には重い |
| **B. 名鉄公式ページのスクレイピング（採用）** | `https://top.meitetsu.co.jp/em/` の運行情報本文を Job が取得・パース | 無料・公式情報。HTML 変化に弱く、サイト利用規約の留意が要る |
| C. 職員が手入力 | 外部連携せず閉域維持 | ユーザーは「外部 API で自動取得」を選択済 → 不採用 |
| D. ODPT | 無料 | 名鉄の運行情報は対象外 → 笠松駅で使えない |

## 決定

**B（名鉄公式ページのスクレイピング）+ 天気と同じ閉域パターン（ADR-021）** を採用する。

1. **取得は backend Cloud Run Job のみ。端末（サイネージ）は DB キャッシュ `railway_status` を読むだけ**で、**名鉄サイトを直叩きしない**（端末側の閉域を維持＝ADR-021 と同思想・[[closed-system-security]]）。盤面の他要素と同じ tx 内で RLS 越しに読む。
2. **外部 egress は Job の単一経路**（VPC connector → Cloud NAT、ADR-009）。weather Job と同じ network 経路・同じ Terraform 構成（`cloud_run_job_railway_status`）。
3. **低頻度・礼儀・fail-soft**: 取得は低頻度（例: 5 分間隔）・**User-Agent 明示**・タイムアウト・キャッシュで名鉄サイトへの負荷を最小化。HTML 構造変化・取得失敗は **last-known-good 表示 or 非表示**に倒し（Sentry 記録）、盤面を壊さない（NFR02）。
4. **`railway_status` は school_id 非保持の公開・非 PII キャッシュ**（運行情報は公開情報）。RLS は weather_forecasts と同じ **`read_all USING(true)` + 書込は system_admin のみ**（ADR-019 §公開参照マスタ特例・ルール6）。**生 PII を列に入れない**（運行情報本文のみ）。取得 Job は system context で upsert。

## 規約・robots（責任実装の前提）

- **robots.txt は 404**（`https://top.meitetsu.co.jp/robots.txt` に存在しない）＝ robots による自動取得の**明示的禁止は無い**。
- ただし**サイトの利用規約（Terms of Use）は robots とは別**であり、自動取得を制限しうる。本方式は **低頻度 + キャッシュ + User-Agent 明示**で負荷・礼儀に配慮するが、**名鉄が規約で自動取得を禁じている / 将来禁じた場合は本方式を停止し、A（商用 API）または C（手入力）へ切替える**（再検討トリガ）。
- スクレイピングは**非公式・無保証**（HTML 構造変化で壊れる）。壊れても fail-soft（盤面は他要素を維持）。

## 残存リスク

- ① **規約**: サイト利用規約で自動取得が制限される可能性（robots は許容だが規約は別）。
- ② **脆弱性**: HTML 構造変化で取得不能になりうる（fail-soft で盤面は壊れない・Sentry で検知）。
- ③ 取得経路の egress が 1 つ増える（ADR-009 の単一 egress 管理下）。

## 再検討トリガ

- 名鉄が公式 API / オープンデータ提供を開始した → 公式 API へ移行（スクレイピング廃止）。
- 名鉄の利用規約で自動取得禁止が明確化した → 即停止し A（商用）または C（手入力）へ。
- HTML 変化が頻発し維持コストが上がった → 商用 API を再評価。
- 商用 API の予算が付いた → A へ移行。

## 影響

- STATUS.md の 2026-05-28「外部取込みしない」を、**運行情報の 1 点に限って更新**（天気=ADR-021 に次ぐ 2 例目の外部取得。その他の外部システム自動取込みは依然として実装しない）。
- 新規 `railway_status` テーブル（公開・非 PII・RLS read_all）+ 取得 Job（`apps/jobs/src/railway-status/`・別 PR）+ Terraform（`cloud_run_job_railway_status`・別 PR）。**端末閉域は維持**（端末は DB のみ読む）。
- ルール4（Vertex マスキング）非接触（運行情報は LLM/embedding 経路に入れない）。
