## 文脈

pattern2/3 のサイネージ盤面に **工学系ニュース** を出したい（ユーザー要望。岐南工業＝工学系学科の生徒向けに適する）。

- 一般時事の**民間ニュース**（NHK / Yahoo! / 時事通信 / 共同通信）は、無料 RSS が技術的に取れても**利用規約で学校サイネージ（公衆送信）に許諾が要る**（NHK「事業・学校利用は別途確認」/ Yahoo「複製・公衆送信・転載しないことに同意」）か、**有料のサイネージ向け配信商品**として販売されている（共同通信デジタル / 時事 FITS BOARD、いずれも台数課金・要見積もり）。NewsAPI 等の汎用 API の無料枠は**本番利用禁止**（dev/test 専用）。
- **工学 / 科学技術ニュース**は発信元が政府系で、無料・自動・適切性の相性が良い。府省（経産省・文科省）は**政府標準利用規約（CC BY 互換）**、JST・JAXA 等は各機関の独自規約（公衆送信は要許諾）。事件事故のような適切性リスクがなく公立校の生徒向けに適する。
- STATUS.md「外部システム自動取込みは現フェーズで実装しない（2026-05-28）」を、天気（ADR-021）・鉄道（ADR-035）に次ぐ **3 例目**として、工学ニュースの 1 点に限り反転する。

## 候補

| 候補 | 概要 | 評価 |
|---|---|---|
| A. 民間時事の有料サイネージ配信 | 共同通信デジタル / 時事 FITS BOARD 等 | 要件は満たすが**有料・台数課金**。MVP 不可 |
| B. 民間ニュースの無料 RSS を本文表示 | NHK / Yahoo! 等 | 規約で公衆送信禁止 / 要許諾。不採用 |
| **C.（採用）政府系 + JST の工学ニュースを「見出し + 出典 + リンク」のみ表示** | 本文非転載 | **無料・許諾不要・合法**。内容は工学系で生徒に適切 |
| D. 教員が記事を都度承認 | 人手キュレーション | 毎日の工数が大きい。ユーザー却下 |

## 決定

**C（見出し + 出典のみ表示 + 天気/鉄道と同じ閉域キャッシュ）** を採用する。

1. **著作権は「事実」を保護しない。** 記事**本文を一切転載せず、見出し（タイトル）+ 発表元 + 公開日 + 出典 URL / QR のみ**を表示する。これにより **Tier1（CC BY: 経産省・文科省）/ Tier2（要許諾: JST・JAXA）を区別せず、全ソースを許諾なし・無料で合法に紹介**できる。**出典明記は必須**（CC BY の条件かつ礼儀）。
2. **取得は backend Cloud Run Job のみ。端末（サイネージ）は DB キャッシュ `news_items` を読むだけ**で、外部 RSS を直叩きしない（ADR-021 / ADR-035 と同じ閉域・単一 egress ADR-009）。
3. **ソース（確認済）**:
   - `jst` = JST サイエンスポータル `https://scienceportal.jst.go.jp/feed/rss.xml`（**本命**・工学/科学技術・日本語・日次更新）
   - `mext` = 文部科学省 `https://www.mext.go.jp/b_menu/news/index.rdf`（CC BY 互換。行政ノイズ多めで補助）
   - `meti` = 経済産業省（RSS 配信は実在。プレス個別 URL は実装時確定）
   - **JAXA は一般プレス RSS が 2021 年終了 / きぼう feed は更新停止（stale）のため MVP 対象外**。
   - フィード URL は env 上書き可（`NEWS_FEEDS_JSON` 等）にして、URL 変化・追加に追従する。
4. **`news_items` は school_id 非保持の公開・非 PII キャッシュ**。RLS は weather_forecasts（0017）/ railway_status（0025）と同じ **`read_all` USING(true) + `write_system`（system_admin のみ書込）**＝ ADR-019 §公開参照マスタ特例。**本文・PII を列に入れない**（見出し + URL + メタのみ）。表示は **pattern2/3 のみ**（pattern1 は対象外）。
5. **低頻度・礼儀・fail-soft**: 取得は 30〜60 分間隔・**User-Agent 明示**・タイムアウト・`(source, url)` で重複排除 upsert。取得失敗 / XML 不正は**当該フィードのみ skip** し last-known-good を維持、盤面の他要素を壊さない（NFR02）。

## 規約・robots（責任実装の前提）

- 各機関フィードは公開 RSS。**本文非転載 + 出典明記 + リンク + 低頻度取得**で礼儀・権利に配慮する。
- JST 等が将来「見出しの転載も禁止」と明確化した場合は当該ソースを停止する（または自前要約 = 事実の言い換えへ切替、§残存リスク ①）。

## 残存リスク

- ① **見出しの著作物性**: 日本の判例上ニュース見出しは原則創作性が低く非保護だが、組織的フリーライドで不法行為が問われた例がある。本方式は **出典明記 + リンク + 低頻度**で正当な「紹介」に留める。グレーが顕在化したら自前要約（事実の言い換え）へ切替える。
- ② **RSS URL / 構造変化・配信停止**: fail-soft + env 上書き + Sentry で検知。
- ③ 取得経路の egress が 1 つ増える（ADR-009 の単一 egress 管理下）。

## 再検討トリガ

- いずれかのソースが利用規約で見出し転載を禁止した → 当該ソース停止、または自前要約へ。
- フィードの構造変化・配信停止が頻発し維持コストが上がった → ソース見直し。
- 「一般時事も出したい」要望が出た → 有料配信（候補 A）の見積もり or NHK 学校利用許諾を別途検討。

## 将来拡張（本 ADR のスコープ外・休眠）

- **Gemini で生徒向け 1 文要約を自前生成**（packages/ai）。要約は事実の言い換えで著作権上クリアだが、事実誤り（ハルシネーション）リスクのため原文リンク併記が前提。ADR-030 PII gate と整合確認の上で導入する。
- **per-school 表示 ON/OFF・ソース選択**（school_configs.display_settings）。初版は pattern2/3 全校で常時表示。

## 影響

- STATUS.md の 2026-05-28「外部取込みしない」を、**工学ニュースの 1 点に限って更新**（天気 = ADR-021、鉄道 = ADR-035 に次ぐ 3 例目）。その他の外部システム自動取込みは依然として実装しない。
- 新規 `news_items` テーブル（公開・非 PII・RLS read_all）+ 取得 Job（`apps/jobs/src/news/`・PR2）+ Terraform（`cloud_run_job_news`・PR2）+ 盤面ブロック（pattern2/3・PR3）。**端末閉域は維持**（端末は DB のみ読む）。
- ルール4（Vertex マスキング）は本体非接触（ニュースを LLM / embedding 経路に入れない。将来の AI 要約拡張でのみ関係）。
