# ADR-028: F06 生徒対話チャットボットの回答ポリシー（スコープ・ハルシネーション境界・多言語・対象）

- 状態: Accepted
- 日付: 2026-06-01
- 関連: [#42](https://github.com/cometa-kaito/kimiterrace-v2/issues/42), ADR-005 (Vertex AI), ADR-006 (Vercel AI SDK), ADR-007 (pgvector), ADR-016 (magic link 匿名), ADR-017 (confidence), ADR-019 (RLS 二層), [CLAUDE.md ルール4](../../CLAUDE.md) (PII マスキング)

## 文脈

F06（生徒対話 Q&A, RAG + SSE）は、生徒が掲示物について質問できるチャットボット。CLAUDE.md / ROADMAP は「チャットボット仕様（誰向け・何を答えるか・トーン・予算枠）は人間が決める」とし、人間ゲートにしていた。

スタック（Gemini Pro / Vercel AI SDK SSE / pgvector RAG / PII マスキング / rate limit）・監査・RLS はすでに ADR-005/006/007/017/019 と [student-qa シーケンス図](../architecture/sequence-diagrams/student-qa.md) で確定済み。データ層（`ai_chat_sessions` / `ai_chat_messages` / `content_versions.embedding`）も実装済み。一方で **回答ポリシーそのもの**（何にどう答え、どこで拒否し、根拠が無い時どう振る舞い、誰が使うか）が未確定だった。

公立高校の生徒が相手で、AI が学校固有の事実（日時・持ち物・場所・対象クラスなど）を誤って答えると、生徒の不利益・学校への不信・賠償リスクに直結する。「便利さ」と「ハルシネーション抑止」のトレードオフを明文化する必要がある。

## 決定

2026-06-01 にユーザー（人間ゲート）が以下を確定した。

1. **対象**: MVP は **生徒 + 教員**。保護者は Phase 2 に送る。
   - 生徒はクラス magic link 経由・匿名（client_id cookie）。教員は Identity Platform 認証済みセッション（rate limit キーは user_id）で既存経路に載る。
2. **回答スコープ**: 掲示物に関する Q&A のみ。学習・進路など掲示物と無関係な話題は、Gemini 呼出前に分類して「ごめんなさい、それは掲示物の話題から外れます」で **誘導なし拒否**（既存方針を維持、コスト + プロンプトインジェクション対策）。
3. **根拠なし時の補足（ハルシネーション境界）**: 掲示物の話題だが掲示に根拠が無い（RAG 非ヒット / 掲示に詳細なし）場合、**関連掲示を案内しつつ一般範囲で補足する**。ただし以下のガードレールを必須とする:
   - 補足には「掲示には無い一般的な情報です」と **明示ラベル**を付ける。
   - 日時・持ち物・場所・対象クラスなど **学校固有の事実は推測で生成しない**。「先生に確認してください」と誘導する。
   - 応答には出典 (evidence_chunk_ids) と confidence を保管する（ADR-017）。
4. **トーン**: 中立・丁寧（敬語ベース、キャラ付けなし）。
5. **多言語**: 主要外国語（やさしい日本語を含めてよい）にも対応する。PII マスキング・スコープ分類・拒否/ラベル文言を多言語で機能させる。
6. **予算**: コスト天井は設けず（NFR06）、不正抑止の rate limit 二層（magic_link 10/分・cookie 10/分）+ Cloud Armor（1000 req/分/IP）で守る。モデルは Gemini Pro 固定（MVP）。

## 検討した代替案

- **根拠なし時を「掲示に記載がありません」で完全拒否（補足なし）**: 最もハルシネーションが少なく安全。却下理由: 「持ち物は？」など掲示の周辺にある自然な質問にすべて沈黙すると UX が悪く、生徒が「使えない」と離れる。安全はガードレール（ラベル + 学校固有事実の推測禁止 + 先生誘導）で担保できると判断。
- **補足を無制限に一般知識で許可**: 最も便利。却下理由: 学校固有事実を捏造するリスクが公立校データ前提では許容できない。
- **保護者を MVP に含める（保護者専用リンク / 認証を新設）**: ニーズはあるが、認証・PII 可視範囲・監査キーの別設計が必要でスコープが膨らむ。MVP の最短到達を優先し Phase 2 に送る。
- **MVP は日本語のみ**: 実装・検証が軽い。却下理由: 公立校は外国籍生徒が一定数おり、初期から多言語を要件に含める（Gemini が多言語ネイティブで増分が小さい）。

## 結果（Consequences）

- **良い影響**: 回答ポリシーが明文化され、実装・レビュー・Phase 検証（トラック① 機能 / ③ セキュリティ）でトレース可能になる。ハルシネーション境界が監査可能な形（ラベル + 出典 + confidence）で固定される。
- **悪い影響 / コスト**:
  - 多言語化により PII マスキング（`packages/ai/src/pii/mask.ts` は日本語前提）・スコープ分類器・拒否/ラベル文言・テストが対応言語ぶん増え、検証面が拡大する。
  - 「根拠なし時の一般補足」は、プロンプト構造で「掲示根拠 ↔ 一般補足」を分離し、ラベル付与と学校固有事実の抑止をプロンプト + 後処理で強制する実装が必要。
  - 教員アクセスは認証経路と rate limit キーが生徒（client_id / magic_link）と別（user_id）になり、エンドポイントが二経路を扱う。
- **トレードオフ**: 「完全拒否（最安全）」ではなく「ガードレール付き補足」を選んだため、ガードレールが破れた時のハルシネーションが残存リスク。**Phase 検証③で敵対的テスト（プロンプトインジェクション + 学校固有事実の誘導）を導入前ゲートとして必須化**する。
- **未確定（実装で決め本 ADR に追補）**: 補足を許す confidence しきい値の具体値、スコープ分類の実装（キーワード + 分類モデル）。値が固まり次第ここに追記する。
